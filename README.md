# Presaler Board

Простейший статический сайт для GitHub Pages без фреймворков и сборки. Это стартовый интерфейс для карты процессов и результатов пресейлов студии Blaster: от входящего брифа до исхода тендерного этапа.

## Что внутри

- `index.html` — структура страницы
- `styles.css` — адаптивные стили
- `script.js` — демо-данные, фильтры, базовая интерактивность

## Что уже умеет страница

- показывает сводку по активным пресейлам, ожиданию решения и win rate
- отображает карту этапов пресейла
- даёт список запросов с поиском и фильтрами
- показывает детальную панель по выбранному пресейлу
- держит всё в чистом статическом формате, подходящем для GitHub Pages

## Публикация на GitHub Pages

1. Добавь файлы в репозиторий и запушь ветку `main`.
2. Открой репозиторий на GitHub.
3. Зайди в `Settings` -> `Pages`.
4. В блоке `Build and deployment` выбери:
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`
   - `Folder`: `/ (root)`
5. Сохрани настройки.
6. Подожди, пока GitHub опубликует сайт.
7. После публикации страница будет доступна по адресу вида:
   `https://liza759.github.io/Presaler/`

## Локальный просмотр

Можно просто открыть `index.html` в браузере. Для GitHub Pages никаких дополнительных инструментов не нужно.

## Следующий логичный шаг

Этот каркас легко расширить до живого борда:

- хранить данные не в `script.js`, а в JSON или через API
- подключить ссылки на Google Drive, Docs, Slides и сметы
- добавить редактирование статусов, дедлайнов и причин победы/проигрыша
- подвязать обновление данных к локальному Codex-процессу на ноутбуке
